# AIスケジュール整理アプリ 要件定義書

**バージョン:** v1.0  
**最終更新:** 2026年3月28日

---

## 1. プロダクトビジョン

### コアコンセプト

やりたいことを入力するだけで、AIが重要度・インパクトをもとに優先順位を視覚的に整理してくれる個人向けスケジュール管理ツール。最終判断はユーザーが行い、AIは補助役に徹する。

### 背景・課題

- スケジュールをうまく組み立てることが難しく、何から始めればよいか分からない
- 管理・記録が面倒で習慣が続かない
- 仕事・プライベート・健康習慣がバラバラで一元管理できていない

## 2. ターゲットユーザー

| 項目 | 内容 |
|------|------|
| サービス種別 | 一般向け（BtoC） |
| 利用シーン | 一人でサクッと使う個人利用 |
| 管理カテゴリ | 仕事・タスク管理、プライベートの予定、健康・運動習慣 |

---

## 3. AI機能仕様

| 項目 | 内容 |
|------|------|
| AIの役割 | 「提案するAI」— 補助に徹し、最終決定はユーザーが行う |
| 優先順位の判断基準 | 重要度・インパクト |
| 提案の表示形式 | 視覚的な並べ替えリスト |
| AIの関与度 | 補助（最終決定はユーザー） |
| ユーザー操作 | AI提案を承認・修正・却下できる |

---

## 4. 機能一覧

### Phase 1 — MVP

| 機能 | 概要 |
|------|------|
| 自然文入力 → AI整理 | メモや箇条書きを入力するとAIがタスク・予定に分類する |
| AI優先順位提案 | 重要度・インパクトを基準に並べ替えを提案。ユーザーが承認・修正・却下できる |
| スケジュールビュー | 1日・1週間単位で整理済みスケジュールを表示。ドラッグ&ドロップで手動調整も可能 |
| 習慣トラッキング | 健康・運動習慣の記録と継続状況の確認 |
| リマインダー通知 | 忘れを防ぐブラウザ通知機能（Web Push） |

### Phase 2（MVP後）

| 機能 | 概要 |
|------|------|
| スマホ対応（PWA） | WebをPWA化してホーム画面に追加、アプリ感覚で使用可能 |
| Googleカレンダー双方向同期 | 既存カレンダーとリアルタイムで同期 |

### 未定

| 機能 | 概要 |
|------|------|
| 収益化機能 | モデル未定（フリーミアム or サブスクを後日検討） |

## 5. 技術スタック

### フロントエンド

| 項目 | 技術 | 備考 |
|------|------|------|
| UIフレームワーク | React | コンポーネントベースのUI構築 |
| ビルドツール | Vite | 高速な開発環境 |
| スタイリング | Tailwind CSS | レスポンシブ対応 |
| ホスティング | Vercel | CDN込み、デプロイが簡単 |

### バックエンド

| 項目 | 技術 | 備考 |
|------|------|------|
| フレームワーク | Django | 認証・管理画面・ORM等の標準機能が充実 |
| API方式 | Django REST Framework（DRF） | REST API、Djangoとの相性◎ |
| 認証 | Simple JWT | JWTトークンによる認証 |
| ホスティング | Railway / Render | 無料枠あり |

### データベース

| 項目 | 技術 | 備考 |
|------|------|------|
| 本番DB | MySQL | DjangoのORMで操作 |
| 開発用DB | SQLite | ローカル開発時に使用可 |

### 認証方式

| 方式 | 実装方法 |
|------|----------|
| メールアドレス + パスワード | Django標準認証 + Simple JWT |
| Googleアカウント | django-allauth または social-django |

### AI連携

| 項目 | 内容 |
|------|------|
| 使用API | Anthropic API |
| 呼び出し元 | Django（バックエンド）側から呼び出し |
| 用途 | タスク分類・優先順位提案 |

## 6. 開発ロードマップ（MVP）

### Step 1 — 環境構築・土台づくり（1〜2日）

- [ ] Djangoプロジェクト初期化（DRF・Simple JWT・django-cors-headers・MySQL接続）
- [ ] Reactプロジェクト初期化（Vite + Tailwind CSS、Axiosセットアップ）
- [ ] CORS設定・API疎通確認

### Step 2 — ユーザー認証（2〜3日）

- [ ] メール認証API実装（登録・ログイン・ログアウト・JWTトークン発行）
- [ ] Google OAuth実装
- [ ] フロントエンド：ログイン・新規登録画面

### Step 3 — タスク・スケジュールのCRUD（3〜5日）

- [ ] DBモデル設計（Task・Schedule・Habit）
- [ ] CRUD APIエンドポイント実装
- [ ] タスク入力・一覧画面
- [ ] スケジュールビュー（1日・1週間、ドラッグ&ドロップ）

### Step 4 — AI機能の組み込み（3〜4日）

- [ ] Anthropic API連携（入力テキスト → タスク分類・優先順位のJSON返却）
- [ ] AI提案UIの実装（承認・修正・却下インターフェース）

### Step 5 — 習慣トラッキング・リマインダー（2〜3日）

- [ ] 習慣記録機能（継続日数・達成率）
- [ ] リマインダー通知（Web Push）

---

## 7. 技術上の注意事項

- ReactとDjangoは別サーバーで動作するため、**CORS設定**が必須
- JWT認証はフロントエンドでトークンを管理する（localStorageまたはhttpOnly Cookie）
- Anthropic APIキーはバックエンド側で管理し、フロントエンドに露出させない
- PWA対応はPhase 2で実施（Next.js移行は不要、Vite + vite-plugin-pwaで対応可能）
